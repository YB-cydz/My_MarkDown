# 北理莫斯科北极熊

## 仿真功能启动顺序

启动仿真世界 -> 启动一些数据处理节点（imu、雷达点云数据处理（地面分割）、pointcloud_to_laserscan的重映射）-> 根据lio参数启动相应的里程计节点（fastlio/pointlio）-> 根据mode模式选择是否根据已有地图导航（其中nav模式下根据参数localization选择使用slam_toolbox/amcl/icp；mapping模式下使用slam_toolbox） -> 启动navgation2

## 实车导航

发布机器人模型（仿真时gazebo的launch做了这个事） ->启动雷达驱动（仿真无需硬件驱动） -> 其余框架同上（主要不同点在于调用的参数文件按sim和real分开了）

# 川大火锅

重定位（包含机器人状态发布、里程计fastlio、重定位icp、rviz可视化以及imu、线速度等的数据处理） -> 导航 -> 决策

无先验地图下的slam，结合`octomap`（用于三维环境建模的开源库）和`autonomous_exploration_development_environment`功能包的地形分析
